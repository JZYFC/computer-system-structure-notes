计算机系统结构

寻址方式

立即寻址：指令直接给出操作数

寄存器寻址：操作数存放于寄存器中

实际上前两种不怎么需要寻址

存储器类寻址：操作数存放于存储器中

注意和模型机的寻址方式相对比

立即寻址

指令中的源操作数是立即数，即源操作数是参加操作的数据本身

例：MOV AX，1200H

立即寻址仅用于源操作数，主要用于给寄存器或存储单元赋初值

寄存器寻址

操作数在CPU的通用寄存器中

例：MOV AX，BX

AX ← BX

8086 为双向内总线, 不用ALU
指令的两个操作数都可以位寄存器寻址

直接寻址

操作数在存储器，指令中直接给出操作数偏移地址

默认在数据段（DS提供段基址）

例：MOV AX，[1200H]

直接寻址注意:

  • 存储器操作数的长度由指令中另一个操作数的长度决定

除了串操作数, 两个操作数不能均为存储器操作数, 此时就可以确定长度
串操作数可以以“修饰”的方法限定

例：

MOV [1234H]，CX
MOV CL，[1234H]

  • 直接寻址方式下，操作数的段地址默认为数据(DS)，但允许段重设，即显示指定数据存放的逻辑段

例：MOV AX，ES:[1200H] ；指定附加段

这种情况称为段超越，指定的段寄存器叫段前缀

寄存器间接寻址

操作数在存储器中，其偏移地址由指令中指定的间址寄存器提供。

例：若DS=1000H

MOV BX，1200H
MOV AX，[BX]

寄存器间址注意:

  • 存放偏移地址的寄存器称为间址寄存器，可用BX，BP，SI，DI，其中BX、BP称为基址寄存器，SI、DI称为变址寄存器

  • 偏移地址长度均为16位，间址寄存器必须是16位寄存器

  • 操作数所在逻辑段与间址寄存器有关：

    • BX, SI, DI → DS 默认在数据段
    • BP → SS 默认在堆栈段

示例
若已知DS=1000H，ES=2000H，SS=3000H，BX=0100H，BP=0056H，SI=0098H，计算下列指令源操作数的物理地址：
MOV AX, [BX] → 10100H
MOV AX, [BP] → 30056H
MOV AX, [SI] → 10098H
MOV AX, ES:[SI] → 20098H

寄存器相对寻址

操作数在存储器，偏移地址为间址寄存器的内容加上一个位移量（常量）

可用寄存器 BX BP DI SI

例：位移量可以为常量
MOV BX，220H
MOV CL，[BX+5]

例：位移量也可为符号地址
MOV AX，[BX+DATA]
MOV AX，[BX]DATA
MOV AX，[BX]+DATA
上面三条完全一样
DATA也可以在BX之前


字母C地址: 30102H

基址-变址寻址

操作数在存储器，偏移地址：

基址寄存器的内容 + 变址寄存器的内容

例，MOV AX, [BX][SI]

操作数段地址取决于基址寄存器

基址寄存器为BX，默认在数据段
基址寄存器为BP，默认在堆栈段

可用变址寄存器: SI DI

基址-变址寻址不能同时用两个基址寄存器或两个变址寄存器，错误示例：

使用示例:

基址-变址-相对寻址

操作数在存储器，偏移地址：

基址寄存器的内容 + 变址寄存器的内容 + 位移量；

隐含寻址

指令中隐含了一个或两个操作数的地址，即操作数在默认的地址中

隐含的操作数既可以在寄存器，也可以在存储器

注意事项

  • 立即寻址只能用于源操作数
  • 指令中的操作数若有[ ]形式，则表示操作数为存储器操作数
  • 可作为间址寄存器的只有4个
    • BX，SI，DI，BP
    • 使用BP时默认在堆栈段，其他为数据段
    • 可以使用段前缀来改变默认的段。

总结

练习

设DS=1000H，ES=2000H，SS=3000H，SI=0010H，BX=0200H，BP=0060H，符号地址VAR1=0050H，VAR2=0082H,说明下列指令源操作数的寻址方式，并计算存储器操作数的物理地址。

  • MOV AX，0100H → 立即寻址
  • MOV AX，SI → 寄存器寻址
  • MOV AX，[0100H] → 直接寻址, 10010H
  • MOV AX，ES:[SI] → 寄存器相对寻址, 20010H
  • MOV AX，VAR1[BX] → 基址变址寻址, 10250H
  • MOV AX, VAR2[BP][SI] → 基址变址相对寻址, 300F2H
